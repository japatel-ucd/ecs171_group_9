Google Inception Network

Naive Inception Model

The Google Inception Network was designed to handle the huge variation of information location within an image. A group of researchers decided that the solution was to have filters with multiple sizes that operate on the same level. Their explanation is further explained in their paper: Going Deeper with Convolutions .To the side is the naïve inception model. It performs convolution on an input with three different sizes of filters. Max pooling is also performed than the outputs are concatenated for the next inception model.


To make the model less computationally expensive, the researchers limited the number of input channels using a 1x1 convolution before 3x3 and 5x5 convolutions. This dimension reduced inception model because known as GoogLeNet (Inception V1). It contained 9 inception models stacked linearly. The entire network contains 27 layers, including pooling. It ends with global average pooling at the end of the last inception model. Below is an image of the architecture.

DenseNet

DenseNet Channel Concatenation

DenseNet builds on the ResNet architecture by setting up each layer to obtain additional inputs from all preceding layers and pass on its own feature-amps to all leading layers. Concentration is used so that each layer receives collective knowledge from all preceding layers. The network can be thinner so a growth rate of k can be used to represent the additional number of channels for each layer.
The animation shows concertation during forward propagation. Each composition layer consists of pre-activation batch normalization, ReLU, then a 3x3 convolutional layer with an output of feature maps for k channels. Further explanation of the DenseNet architecture can be found in the paper Densely Connected Convolutional Networks.
